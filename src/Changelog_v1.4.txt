Lufia 2 Auto Tracker v1.4 - Changelog
=======================================

Core Changes
------------
*   **Framework Migration**: Ported entire application from Tkinter to PyQt6.
*   **Architecture**: Refactored into a Domain-Driven Design:
    *   `LogicEngine`: Handles accessibility rules and item requirements (decoupled from UI).
    *   `StateManager`: Central source of truth for Inventory, Locations, and Characters.
    *   `DataListener`: Background TCP listener for external auto-tracking data.

UI Improvements
---------------
*   **Docking System**: implemented a flexible DockWidget system permitting users to rearrange panels.
    *   Restored "T-Shape" layout default (Characters top, Map/Tools split bottom).
    *   Added "Fluid" resizing policy allowing the Map to take priority when resizing.
*   **Customization**:
    *   **Header Colors**: Added Options menu to pick custom colors for Dock titles.
    *   **Font Size control**: Added global font scaling buttons (+/-) to dock headers affecting all inner content.
*   **Map Widget**:
    *   Smooth scaling with QGraphicsView.
    *   Context Menus on Right-Click (Item Search for Cities, Character Assign for Locations).
*   **Inventory**:
    *   Used `FlowLayout` for Tools/Keys to allow wrapping.
    *   Reverted grayscale icons to low-opacity (15%) for better visibility.

New Features
------------
*   **Auto-Tracking**: Integration for receiving game state via TCP (e.g. from USB2SNES helper).
*   **Item Search Dialog**:
    *   Standalone window (non-modal or modal options).
    *   **City Picker**: Dropdown to quickly switch target shops.
*   **Character Assignment**:
    *   Logic to assign characters to locations (e.g. "Found Guy at Alunze").
    *   Visual sprites appear on the map at the assigned location.
*   **Persistence**: Full Save/Load State functionality using JSON.

v1.4.1 Update - Free Placement & Polish
---------------------------------------
*   **Free Placement System**:
    *   **Edit Layout Mode**: Enabled via Options > "Edit Layout". Allows simple drag-and-drop of ANY icon in Characters, Maidens, Tools, and Keys widgets.
    *   **Layout Persistence**: Custom positions are saved automatically to `layout_config.json` and restored on launch.
    *   **Scrollable Docks**: Characters widget now uses a ScrollArea, allowing it to be resized smaller than its content.
*   **Auto-Tracking**:
    *   **Granular Filters**: Added toggles in the "Auto" menu to enable/disable tracking for specific categories (Chars, Tools, Keys, Maidens, Position).
*   **Map**:
    *   **Player Color**: Added "Pick Player Color" option to customize the tracker arrow.

v1.4.2 Update - Map & Marker Enhancements
-----------------------------------------
*   **Player Marker**:
    *   **Size Control**: Adjustable player marker size (1x to 4x) via "Tracker > Player Size".
    *   **Sprite Mode**: Option to use the Active Party Leader's sprite as the player marker ("Tracker > Player Shape").
    *   **Logic**: Sprite marker automatically updates when the in-game party leader changes (via auto-tracking).
*   **Map Fidelity**:
    *   **Improved Scaling**: Implemented `SmoothPixmapTransform` to reduce pixelation artifacts when resizing the map.
    *   **Responsive Resizing**: Fixed issue where map contents did not scale correctly with the window.
    *   **Capsule Logic**: Implemented accurate order-based mapping for Capsule Sprites (matching v1.3 logic).
*   **Layout Refinements**:
    *   **Dock Sizing**: Constrained Items and Hints widgets to prevent them from consuming excessive screen space.
    *   **Manual Assignment**: Restored context menus for manually assigning characters to locations.

v1.4.3 Update - Emulator Compatibility
--------------------------------------
*   **Expanded Emulator Support**:
    *   **Snes9x-nwa**: Added support for split-memory bank layouts (Dungeon Flags at 0xE2...).
    *   **bsnes**: Verified compatibility.
*   **Robust Scanning**:
    *   New "Dual Scan" logic dynamically identifies WRAM and ROM regions regardless of emulator memory model.
    *   Sprite Table scanning ensures correct ROM mapping even in Headerless/Heap-based environments.
*   **Fixes**:
    *   Fixed issue where "Cleared Dungeons" were not detected on NWA versions.
    *   Improved config file loading reliability.

v1.4.5 Update - UI Refinements & Fixes
------------------------------------------
*   **Quality of Life**:
    *   **Scalable Windows**: Converted Maidens and Characters panels to QGraphicsView, allowing contents to scale gracefully when their Dock borders are resized.
    *   **Icon Sizing**: Added dedicated (+/-) scaling buttons for picture sizes across all icon-based widgets, including Characters, Maidens, Tools, and Keys.
    *   **Recover Widgets**: Centralized accidentally closed docks into a custom 'Restore Closed Windows' menu options list.
    *   **Floating Widgets**: Resolved a bug where undocked, floating panels failed to save and restore their screen positions upon restarting the tracker, and ensured they close fully when the main application shuts down.
    *   **Location Text Toggle**: You can now toggle the text labels identifying where characters are found on/off inside the Custom menu.
    *   **City Search Bar**: Formatted the Item Search city dropdown as an editable Auto-Complete typing field.
    *   **Auto Tracking Menu**: Transferred the Auto Checkboxes out of the top ribbon into a dedicated `Tracker` -> `Auto` sub-menu.
    *   **Cleaned Dropdowns**: Purged unreachable game locations (Gordovan, Agurio, Pico Woods) from Item Select tables.
*   **Performance Optimizations**:
    *   **Positional Caching**: Heavily trimmed down Auto-Tracking positional update packets. The tracker now sends a minimal coordinate payload rather than re-computing the entire game inventory every frame a player moves, drastically reducing Python parsing overhead.
*   **Auto-Tracking & Stability**:
    *   **Duplicate Sprite Placements**: Resolved an issue where characters and capsules from auto-tracking could mistakenly duplicate across multiple map locations simultaneously.
    *   **Memory Parsing Fix**: Fixed a critical 64-bit to 32-bit truncation issue in the C# Tracker Helper that caused memory offsets to fail or crash unexpectedly.
    *   **Connection Resilience**: Upgraded tracking sockets to utilize explicit reuse flags (`SO_REUSEADDR`), ensuring rapid tracker restarts do not trigger Windows port blocked errors.
    *   **UI Freeze Safety**: Guarded Python data streams with safe traceback loggers, preventing the main UI from permanently hanging "Scanning in process..." during an unexpected C# helper crash.
